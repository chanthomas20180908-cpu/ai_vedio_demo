# Product Spec - Story Agent（自然语言优先）
## 1. 产品目标
用户只用自然语言即可完成：选素材 →（按需）总结 → 出点子 → 写故事 → 审阅 → 迭代。
`@select/@summary/@ideator/@writer/@reviewer` 仅作为“高级用户快捷方式”，不是必需。
## 2. 核心体验原则
1) 自然语言优先：用户说“我想要什么”，系统负责匹配意图并补齐参数。
2) 缺参就问：范围/题材/字数/风格等缺失时，只追问，不擅自执行。
3) 关键节点确认：执行任何会产生外部效果的动作前，需要用户确认；选 idea 编号、是否采纳 reviewer 建议也需要用户明确选择。
4) 输出可见：每一步都在界面展示关键结果；writer 必须展示 story.md 文件路径。
## 3. 支持的用户意图（MECE）
A. 查询 summaries
- 例句："现在有哪些 summary 题材？" / "列出 summaries" / "summary 都有什么主题？"
- 期望输出：题材清单（题材名 + 主题 1 行 + 文件路径）。
B. 更新 summaries（指定范围）
- 例句："把 ./a.md 和 ./b.md 做成 summaries" / "对这些素材生成 summary"
- 必要输入：范围（files/items/selected）。
- 期望输出：summaries_written + index 路径。
C. 生成点子（ideas）
- 例句："用 克苏鲁 + 段子，给我 3 个恐怖开场但结尾搞笑反转的点子" / "出 3 个方向让我选"
- 必要输入：summaries 范围（all 或指定题材）。
- 期望输出：3 个 ideas（编号 + title + angle）。
D. 写故事（writer）
- 例句："选第 2 个点子写成 800 字" / "开始写一个短篇，恐怖但最后反转搞笑"
- 必要输入：idea_index（若未生成 ideas，先进入生成点子流程）。
- 期望输出：story.md 文件路径（必须展示）。
E. 审阅与迭代
- 例句："审一下" / "给我 3 条最关键修改建议" / "采纳建议重写一版"
- 期望输出：issues/suggestions（精简）→ 用户选择是否采纳 → 进入重写循环（v002/v003...）。
## 4. 用户输入最佳实践（黄金句式）
1) 查询 summaries："现在有哪些 summary 题材？"
2) 生成点子："用【克苏鲁+段子】做一个【恐怖开场、结尾搞笑反转】的小故事，先给我 3 个点子。"
3) 写作："选第 N 个点子，写成【800-1200 字】，【第一人称】，【结尾一句金句】。"
4) 审阅："审一下，给我 3 条最关键修改建议。"
5) 迭代："采纳建议，重写：更恐怖、更好笑、反转更突然但逻辑自洽。"
