# 目标
- 多轮对话中：**可选原文、可改故事、可再选**。
- **最终故事是重新生成**（可吸收原文信息，但不直接拼接）。
- 保留**可追溯性**（知道哪些原文影响了哪些内容）。

---

# 核心设计（最小方案）

## 1) 多轮对话的“状态结构”
每次对话保存一个“session 状态”，避免复杂 agent 框架。

**sessions/<session_id>/**
- `state.json`：当前对话状态  
  - `selected_passage_ids[]`：当前选中的原文片段  
  - `story_goal`：本轮想要的故事目标/风格  
  - `constraints[]`：必须保留的事实/主题  
  - `latest_story_path`：最新生成的故事文件  
- `turns/turn_001.json`：每一轮的 user/assistant 输入输出摘要
- `story/story_v001.md`：生成故事版本
- `story/story_v001.meta.json`：记录“来源原文id + 影响链”

> **关键点**：多轮不是靠“拼接”，而是通过**状态 + 选择 + 约束**驱动重新生成。

---

## 2) 原文选择（Selection）
- Agent 提供：**检索 + 预览 + 选择**
- 选择只保留“候选事实/素材”，而非直接拼接输出。

**推荐最小功能**  
- `kb-list-passages --q --tag --source-id`  
- `select --add / --remove / --clear`  

---

## 3) 生成逻辑（不是拼接）
- “生成故事”的输入应是：
  1) `story_goal`（目标故事）
  2) `selected_passage_ids` 对应的“素材摘要”
  3) `constraints`（硬性事实 / 禁止偏离）

- 输出是**全新故事文本**，但**事实必须在素材范围内**（防幻觉）

**生成策略极简化**：
- 先把选中的原文变成**素材要点摘要**（不可改写事实）
- 再让模型用这些要点生成故事正文

---

## 4) 修改机制（再次生成）
用户的修改分两类：
1) **改故事**（比如“更黑暗”“更简短”）  
   → 更新 `story_goal` 或 `constraints`  
2) **改素材**（比如“换掉这两段原文”）  
   → 更新 `selected_passage_ids`

然后重新生成新版本 `story_v002.md`

# 最小交互流程（多轮）
```
用户：我想做一个克苏鲁式心理悬疑故事
Agent：好的。请先选原文素材（我可以帮你检索）
用户：选择 passage A,B,C
Agent：收到。你希望故事风格/长度/结构？
用户：短篇、压抑、第一人称
Agent：生成 story_v001.md
用户：结尾太快，改成开放式
Agent：更新约束，生成 story_v002.md
用户：再加入 passage D
Agent：更新选集，生成 story_v003.md
```

# 关键点保证“不是拼接”
- **禁止直接拼接原文**作为正文（只能引用事实）
- 素材仅用于“事实和灵感”，**输出必须重写**
- 仍保留 `meta.json` 记录来源，保证可追溯

# Warp 交互规范（对话编排）
本规范用于在 Warp CLI 中完成“伪 agent”编排，**不新增脚本、不改底层角色逻辑**，只通过对话驱动现有脚本执行与结果展示。

## 1) 交互原则
- **全程对话驱动**：用户一句话 → 生成执行预告 → 用户确认 → 执行脚本。
- **关键输出可见**：每一步都要在界面输出关键结果摘要。
- **最小改动**：不新增 prompt / workflow 配置文件。

## 2) 执行预告（固定三行）
1) 目标：我要产出什么  
2) 动作：我将运行哪些命令/做哪些改动  
3) 产出：结果会保存到哪里/会生成哪些文件  
用户回复“执行/跑/开始”后才允许执行。

## 3) 关键输出格式（对话展示）
- **select**：输出当前选中 items 列表  
- **reader**：输出每个 source 的概览（source_label / source_theme / facts 数 / motifs 数 / conflicts 数）  
- **ideator**：输出 ideas 列表（编号 + title + angle），**可选**  
- **writer**：仅输出 `story.md` 文件路径  
- **reviewer**：输出 issues / suggestions 列表（各 3 条），**可选**  

## 4) 选择与循环
- ideator / reviewer 输出后，用户直接回复编号选择  
- writer-reviser 循环：reviewer 结果 → 生成改写指令 → 作为 writer 的 `--input` 继续下一轮
